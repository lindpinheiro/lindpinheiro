Passo a passo
Plano de migração do Products Fruits
⚠
O Products Fruits não exporta dados históricos de engajamento via API pública. O que migra é o conteúdo (textos, lógica dos tours) — não o histórico de quem completou o quê.
1

Inventário dos tours e modais existentes

Liste todos os fluxos ativos: nome, URL gatilho, segmento alvo, sequência de passos e métricas de referência disponíveis no dashboard atual.

Estimativa: 1–2 dias · responsável: produto
2

Instalação em staging + configuração de usuários

Instalar o snippet em staging. Configurar o identify(userId, { plano, role, dataCadastro }) — essa etapa define a qualidade da segmentação futura.

Estimativa: 1–3 dias · responsável: dev
3

Migração paralela (2–4 semanas)

Rodar ambas as ferramentas simultaneamente. Recriar os flows prioritários e validar o comportamento. Não cancelar o Products Fruits ainda.

Atenção: custo duplo neste período — negocie mês avulso no PF
4

Instrumentação de eventos customizados

Mapear e implementar track() nos pontos-chave do produto: ações importantes, features visitadas, marcos de onboarding. Define a riqueza dos dados.

Estimativa: 3–5 dias · responsável: dev + produto
5

Go-live e cancelamento do Products Fruits

Validar todos os flows, confirmar dados chegando, cancelar o PF após o próximo ciclo de faturamento. Documentar a nova estrutura para o time.

Marco: migração completa

Perguntas para fazer aos fornecedores
🗺️ Tours e fluxos
  • O time de produto consegue criar tours sem ajuda de dev?
  • É possível criar condicionais dentro de um tour (pular etapa se usuário já fez X)?
  • Como funciona o versionamento? Tem draft vs publicado?
  • Qual o limite de tours ativos no plano contratado?
📊 Analytics e rastreamento
  • O que é rastreado por padrão sem instrumentação adicional?
  • Como vejo quais usuários completaram um flow específico? Tem exportação?
  • Posso rastrear cliques em botões fora dos flows (botões nativos do produto)?
  • Os dados ficam disponíveis por quanto tempo no plano?
🔌 Integração e migração
  • Tem suporte para migração do Products Fruits?
  • Como funciona a integração com Segment/Mixpanel? É bidirecional?
  • Tem webhooks de "flow completado" em tempo real?
  • O snippet impacta a performance de carregamento do produto?
💰 Contrato e suporte
  • O preço escala por MAU ou por usuários identificados? Como é contado?
  • Vocês fazem contrato em reais? Qual a variação cambial?
  • Tem contrato anual obrigatório ou tem opção mensal?
  • Existe onboarding dedicado para novos clientes? Qual o prazo para go-live?
