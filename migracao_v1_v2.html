Os 3 Pilares para Viabilizar a Migração
Funcionalidades essenciais que o V2 precisa entregar para os Pequenos Negócios
1
Funcional

Workspace

Para pequenos negócios com 1-2 locais, o modelo atual de workspace do V2 já é adequado. Não exige mudanças drásticas na visão do usuário.

  • Estrutura de locais existente
  • Visão operacional básica
  • Sem edição em massa necessária
2
Alta prioridade

Monitoramento

O pilar mais complexo e que demandará mais tempo. É o coração do novo modelo operacional baseado em automação.

  • Dashboard de status das automações
  • Alertas e logs de execução
  • Rastreabilidade de dados (crawlers)
3
Alta prioridade

Postagens

Automação de postagens via IA (integração Instagram). O cliente configura uma vez e o sistema executa, eliminando login diário.

  • Automação de postagens via IA
  • Integração com Instagram (V2)
  • Configuração única — execução contínua
Ordem de Prioridade Técnica
Sequência definida em reunião — do maior para o menor esforço estimado
1
Monitoramento
● Alto esforço
Maior complexidade técnica. Requer construção de dashboards, sistema de logs, alertas e rastreabilidade de dados (crawlers externos para verificar consistência com Google).
2
Integração com Instagram
● Médio esforço
A funcionalidade existe no V1, mas precisa ser adaptada para o V2. Exige mapeamento e re-implementação do fluxo de automação de postagens.
3
Listagem
● Menor esforço
Considerada a etapa mais tranquila. Funcionalidade mais direta, menor impacto técnico no processo de migração.
Novo Modelo Operacional
A migração transforma a forma de trabalhar da operação

ANTES — V1

  • ✕ Execução manual empresa por empresa
  • ✕ Time dividido entre V1 e V2
  • ✕ Dependência de listas externas (Monday)
  • ✕ Login diário necessário pelo cliente
  • ✕ Aprovação manual de alterações
→

DEPOIS — V2

  • ✓ Configuração única → execução automática
  • ✓ Time unificado no V2
  • ✓ Controle centralizado na plataforma
  • ✓ Self-service: cliente configura, IA executa
  • ✓ Avisos claros → cliente assume responsabilidade

Desafios e Riscos Identificados
!

Limitações atuais na V2

Dev

Admin não consegue criar locais nem editar planos sem intervenção externa. Necessário resolver antes da migração.

!

Contratos ativos (V1)

Comercial

Clientes têm contratos prometendo funcionalidades que talvez não vão ao V2. Necessário aditivo contratual.

!

Descontinuação de integrações

Produto + Jurídico

Facebook, Bing e Apple Maps: baixa relevância ou sem suporte no Brasil. Foco no Google + IA. Requer alinhamento jurídico.

!

Modelo híbrido (2 públicos)

Marketing + CS

V2 atenderá pequenos negócios e gestores Google. Comunicação e operações precisam ser adaptadas para cada perfil.

!

Confiabilidade de dados

Dev

Discrepâncias entre perfil Google e dados públicos. Precisa de monitoramento externo (crawlers) para garantir precisão.

!

Reeducação do cliente

CS + Produto

Self-service exige mudança de comportamento: configura uma vez e o sistema executa, sem necessidade de login diário.
